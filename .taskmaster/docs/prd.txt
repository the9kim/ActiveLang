## 1. 제품 개요

- 기억 인출 기반의 능동적 어학 학습 지원
    
- 기존의 수동적 단어 암기 방식(단어-뜻 매칭)에서 벗어난 학습 경험 제공
    
- 사용자가 직접 생각하고 표현하며 반복적으로 복습할 수 있도록 설계
    
- 장기 기억 정착과 실질적 활용 능력 향상에 초점
    

## 2. 타깃 사용자

- 영어를 일정 수준 이상 학습했으나 말하기·표현력에 어려움을 느끼는 성인
    
- 반복 학습의 필요성은 느끼지만 기존 단어장 앱에 흥미를 못 느낀 사용자

- 실용적인 문장 암기를 통해 어학 시험·회화·비즈니스 회화 등을 준비 중인 학습자
    

## 3. MVP 목적

- 기억 인출 기반 학습 방식의 수요 존재 여부 
    
- 기존 단어 암기 위주의 앱과 차별화된 사용자 경험 제공 가능성 확인
    
- 초기 사용자 반응을 기반으로 확장 기능 개발 여부 판단
    

## 4. 핵심 기능

- 문장 등록 기능 (영문 문장 + 한글 번역)
    
- 복습 시 한글 번역 제공 → 사용자로 하여금 기억 인출 유도
    
- 인출 후 실제 영문 정답 확인 가능
    
- 사용자가 스스로 학습 결과 평가 (상/중/하)
    
- 평가 결과에 따라 복습 일정 자동 조정 (Spaced Repetition 반영)
    

## 5. 사용자 시나리오

1. 사용자가 외우고 싶은 문장과 번역 입력
    
2. 일정 시간 경과 후 복습 알림 도착
    
3. 한글 번역 제시 → 머릿속 또는 말로 문장 인출
    
4. 인출 후 실제 영문 확인 → 사용자가 직접 학습 수준 평가
    
5. 평가 결과에 따라 다음 복습 일정 자동 설정
    

## 6. 비포함 기능 (향후 고려 대상)

- 음성 인식 기반 말하기 학습
    
- 원어민 음성 제공 기능
    

## 7. 기술 스택 및 아키텍처

- MVP는 Flutter + Supabase 기반으로 개발
    
- 빠른 프로토타입 제작과 사용자 피드백 수집에 최적화
    
- 사용자 확보 및 피드백 수렴 후 Java(Spring Boot) + AWS 기반으로 확장 계획
    
- 장기적으로는 백엔드 확장성, 보안성, 복잡한 비즈니스 로직 대응 고려
    

## 8. 개발 일정 (예상)

|주차|작업 항목|
|---|---|
|1주차|요구사항 정리, 화면 설계, Supabase 초기 세팅|
|2주차|문장 등록/조회 기능 구현|
|3주차|복습 로직 구현 (Spaced Repetition 알고리즘 적용)|
|4주차|평가 시스템 및 복습 일정 자동화 구현|
|5주차|UX 개선, 오류 수정, 테스터 모집|
|6주차|피드백 수집, 버그 수정, 향후 개선 방향 도출|

## 9. 측정 지표 (초기 성과 지표)

- 사용자당 평균 등록 문장 수
    
- 일간/주간 복습 완료율
    
- 학습 지속률 (3일, 7일, 14일 기준 리텐션)
    
- 사용자가 직접 평가한 '상' 비율의 변화 추이
    
- 피드백에서 나타난 학습 효과 실감 여부
